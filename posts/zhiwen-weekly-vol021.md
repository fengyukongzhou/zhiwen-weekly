## 🐎 跑马场

### 🖋️ 逗号

![zw021-002.png](zw021-002.png)

我坚信创作者有权根据个人需要自由地使用标点。有无逗号会改变这个句子的阅读节奏，而作者完全有权做这个决定。是否使用逗号取决于前后语句的节奏，作者的惯常风格，他想要呈现的效果，或天气以及星象之类不可捉摸的因素。最终决策者不应是受过英文写作训练、拿着红色铅笔的文字编辑。

<p align="right">——劳伦斯·布洛克《小说的八百万种写法》</p>

逗号是个语法符号，但是我把它用作一种呼吸的手段。读者绝不可从催眠中醒来。

<p align="right">——《马尔克斯：最后的访谈》</p>

世界语言最重要的是节奏。我们学西方的长句子造成很多人不知道应该在哪儿断句，不知道怎么断。我采取笨方法，标点符号不仅仅是语法的作用，同时是节奏点的作用。

<p align="right">——阿城《闲话闲说》</p>

只有真正的行家才会注意到这一点。我不妨补充一句：英文用逗号是为了文法，中文用逗号是为了文气。

<p align="right">——余光中《变通的艺术》</p>

约二十年前，我有很久没有写中文，一直在念英文，写一点点英文，来港后把旧作整理，出了一本散文集。友人宋悌芬兄看了说：“你的句子太长。”这句话一点不错。我发现我的逗号用得太少，由此悟到中英文标点最大不同点之一，就是英文的逗号用得比中文少，因此把英文译成中文，不得不略加一些逗号。

<p align="right">——思果《翻译研究》</p>

### 🚌 公共汽车

![zw021-003.png](zw021-003.png)

在迪恩·墨菲的访谈中，她说：“我无法想象诗人不去争取安闲和平静。不幸的是，诗歌并非诞生于喧闹、人群之中，也并非诞生于公共汽车上。所以，必须有四面墙，并且保证电话不会响起。这是写作所需要的一切。”

<p align="right">——辛波斯卡《我曾这样寂寞生活》</p>

公共汽车上坐我前面的姑娘，只用根黑头绳扎头发，穿略大的工装衣裙，没有首饰化妆，没穿耳洞。侧脸上的轮廓，是天工一时灵感，没法复刻，近透明皮肤下透出淡蓝血管。看窗外时，像第一次看见世界，叫人以为她是刚刚从哪里来的。这个形象既被最大简化又极其丰富，我对她一无所知，却像坐在教堂里。

<p align="right">——贾行家《潦草》</p>

很长时期，我少年、青少年时期、青年时期，搭公交车时，若不是很拥挤，我都喜欢坐到最后一排靠窗座，这或许可以做精神分析——“年轻时的我，自我意识是画面之外的，多余的那个人”、 “我害怕被若有另一个观察者观察”。也就是说，我害怕在人群中被暴露出来，希望躲成那个叠在墙上的影子。

<p align="right">——骆以军《肥瘦对写》</p>

### 🎨 潦草绘画 Prompt

![zw021-005.png](zw021-005.png)

这段提示词借着 GPT-Image-2 的东风，前段时间比较流行。是把图片转绘成鼠标画图的那种风格。

```
用最笨拙、最潦草、最可悲的方式重新绘制这张图片。使用白色背景，让它看起来像是用滑鼠在 MS Paint 里画的。它应该隐约相似，但又不太像。有点对得上，但又莫名地偏差，带着那种低画质、一个像素一个像素慢慢画的感觉，真正强调出它究竟有多荒谬地糟糕。算了，你知道吗，随便啦，你想怎么画就怎么画吧。
```

我略微发挥了一下，可以将现成画作转换为“写生”的场景，但是画得很潦草。

```
**场景描述**：一个电影质感的、极致写实的艺术工作室，柔和的北欧光线穿过窗户，空气中飘浮着微小的尘埃。在背景中，一位极其专业的模特正在全神贯注地摆出参考图的姿势，服装、妆造和神态完美还原原作的神韵。

**主体焦点**：采用过肩镜头，焦点聚焦在前景画架上的数码绘图板上。画家的手正握着笔在上面创作，而画布上呈现的内容极其荒谬：**[用最笨拙、最潦草、最可悲的方式重新绘制这张图片。使用白色背景，让它看起来像是用滑鼠在 MS Paint 里画的。它应该隐约相似，但又不太像。有点对得上，但又莫名地偏差，带着那种低画质、一个像素一个像素慢慢画的感觉，真正强调出它究竟有多荒谬地糟糕。算了，你知道吗，随便啦，你想怎么画就怎么画吧。]**

**视觉风格**：照片级的写实环境与极度廉价的 MS Paint 像素美学形成剧烈视觉冲突。画室细节达到 8K 分辨率，背景模特带有细腻的浅景深模糊（Bokeh），而画布上的涂鸦则清晰、生硬且带有锯齿。这种“殿堂级表现”与“灾难级创作”的对比创造出一种超现实的幽默感。 --ar 16:9
```

### 📱 微信公众号排版（续）

![zw021-006.png](zw021-006.png)

初版本（见 [Vol.020 辞典排版吃垃圾](/2026/04/24/zhiwen-weekly-vol020/)）还是很潦草的提示词，用于窗口AI。

- 为了处理图片，需要先发博客（相当于图床了），然后用链接引用图片。
- 为了粘贴到公众号编辑器，还需要预览 HTML 后全选复制。

于是用 Agent 改写成了 Skill，Skill - wechat-formatter，工作流大致如下：

1. 读取源文件
2. 提取封面图色板 
3. 解析 MD
4. base64 嵌入图片 
5. 生成 HTML（含 CSS） 
6. 输出至文件夹 `_inbox/` 
7. 浏览器打开

这几步我都不确定具体该用什么方法实现，我只是提出需求，让 Agent 先搜索解决方案，然后动手修改的。

比如，现在的复制方式是，在生成的 HTML 页面上方添加“一键复制”按钮。现在的配色方案是分析文章头图得到的。我不知道实现原理是什么，也不想知道。

有种异样的感觉，但也不是不能接受。

目前已有一套完整的文章后处理 Skill 组合了：

首先是 Skill - rename-linked-images，用于图片附件处理。可以批量重命名笔记引用的图片，并且同步更新其他笔记中的引用链接。还可以转换 Wiki 链接和标准 Markdown 链接的图片引用格式，因为我的笔记里用 Wiki 链接格式，但 Hexo 需要标准 Markdown 链接格式。这个流程之前是写 Obsidian 插件实现的，效果还不够好。

然后用 Skill - hexo-publisher 来发布博客。这个技能很简单，只是转换一下格式——套用 Hexo 的 YAML，归入现有分类、加入现有标签，然后把笔记与附件复制到博客所在文件夹，最后在文件夹目录中运行 Hexo 命令。

最后就是前面提到的 Skill - wechat-formatter，简单排版公众号文章，之后一键粘贴到编辑器即可。

另外，我还发现用 OpenCode 配 DeepSeek V4 Flash 之类的模型处理后，所有中文引号都会在模型“无意识状态”下改为英文引号。所以需要打个补丁：在 Skill 中强调输出时使用直角引号，处理完之后执行替换操作。
